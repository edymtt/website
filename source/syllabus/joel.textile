h3. Joel Test

I've thought to "evaluate myself" with the "Joel Test":http://www.joelonsoftware.com/articles/fog0000000043.html, in order to aknowledge my skills in the things required to run/work in a software company. Enjoy ;-)

<ul>
<li class="check">Do you use source control? Yes</li>
<li class="check">Can you make a build in one step? Yes</li>
<li class="cross">Do you make daily builds? No</li>
<li class="cross">Do you have a bug database? No</li>
<li class="cross">Do you fix bugs before writing new code? No</li>
<li class="cross">Do you have an up-to-date schedule? No</li>
<li class="cross">Do you have a spec? No</li>
<li class="check">Do programmers have quiet working conditions? Yes</li>
<li class="cross">Do you use the best tools money can buy? No</li>
<li class="cross">Do you have testers? No</li>
<li class="cross">Do new candidates write code during their interview? No</li>
<li class="cross">Do you do hallway usability testing? No</li>
</ul>
